# Operations Manual

Command reference for building, running, and debugging.

## Prerequisites

- Rust 1.90.0 (see `rust-toolchain.toml`)
- `cargo install cargo-component --locked` (for building wasm appraisers)
- `rustup target add wasm32-wasip1`
- `openssl` (for generating ES256 key pairs)

## Common Scripts

| Script | Purpose | Dependency |
| ----------------------------- | ---------------------------- | ------------------------------ |
| `scripts/run-mvp.sh` | mock mode end-to-end (no TEE dependency) | — |
| `scripts/gen-keys.sh` | Generate ES256 key pair into `config/keys/` | openssl |
| `scripts/build-appraisers.sh` | Build all wasm appraisers | cargo-component, wasm32-wasip1 |

`config/keys/` is generated by scripts and gitignored.

End-to-end test steps for each TEE require the corresponding hardware environment. See per-TEE docs:

- CCA / CCA + hydra: [cca.md](cca.md)
- Hygon CSV / CSV + hydra: [csv.md](csv.md)
- TDX / TDX + hydra: [tdx.md](tdx.md)
- iTrustee / iTrustee + hydra: [itrustee.md](itrustee.md)
- VirtCCA / VirtCCA + hydra: [virtcca.md](virtcca.md)

### iTrustee end-to-end test

```bash
bash scripts/gen-keys.sh
bash scripts/build-appraisers.sh
cargo build --release -p verifier -p attester -p relying-party

ttrpc-aa &
api-server-rest --features attestation &

./target/release/verifier --config config/verifier-itrustee.toml > /tmp/verifier-itrustee.log 2>&1 &
./target/release/attester --config config/attester-itrustee.toml > /tmp/attester-itrustee.log 2>&1 &
sleep 2

./target/release/relying-party \
    --attester http://127.0.0.1:9000 \
    --verifier http://127.0.0.1:8080 \
    --tee-type itrustee \
    --pubkey config/keys/ear_public.pem \
    --ear-out /tmp/ear-itrustee.jwt
```

## Passport mode

Passport mode is enabled via `--mode passport` on relying-party. The attester must have `verifier_endpoint` configured so it can call the verifier internally. The RP then receives an EAR directly from the attester without calling the verifier.

Passport is independent from hydra: passport only affects the wasm evidence flow over gRPC; hydra runs on a separate TCP subsystem — see [hydra.md](hydra.md).

```bash
./target/release/verifier --config config/verifier.toml > /tmp/verifier.log 2>&1 &
./target/release/attester --config config/attester.toml > /tmp/attester.log 2>&1 &
sleep 2

./target/release/relying-party \
    --mode passport \
    --attester http://127.0.0.1:9000 \
    --tee-type mock \
    --pubkey config/keys/ear_public.pem \
    --ear-out /tmp/ear-passport.jwt
```

The RP validates EAR signature + freshness (iat within 5 minutes) instead of comparing eat_nonce.

## Hydra mode

All three peers must stay running with persistent TCP connections:

```bash
# 1. verifier (config/verifier-cca-hydra.toml already contains [hydra])
./target/release/verifier --config config/verifier-cca-hydra.toml > /tmp/verifier-hydra.log 2>&1 &

# 2. relying-party with --hydra-listen enables the TCP listener
./target/release/relying-party \
    --hydra-listen 127.0.0.1:7002 \
    --hydra-data-dir workspace-data/relying-party \
    hydra-serve > /tmp/rp-hydra.log 2>&1 &

# 3. attester auto-bootstraps when [hydra] is present in its config
./target/release/attester --config config/attester-cca-hydra.toml > /tmp/attester-hydra.log 2>&1 &
sleep 130   # at least one batch window (default 120s)

# 4. one-shot: build the EvidenceReply from the latest session and ship it
./target/release/attester --config config/attester-cca-hydra.toml \
    hydra-evidence --rp 127.0.0.1:7002
```

Key paths:

| File | Role |
| ---- | ---- |
| `workspace-data/verifier/verifier_key.bin` | Verifier key. Losing it invalidates every already-shipped EvidenceReply. |
| `workspace-data/attester/attester_key.bin` | Attester key |
| `workspace-data/attester/attester_latest_session.txt` | Pointer used by `hydra-evidence` when `--session` is omitted |
| `workspace-data/attester/attester-runs/attester-<time>-<pid>-<counter>/` | Per-session dev_infor / dev_res / dev_config / public_context |
| `workspace-data/relying-party/public_context.bin` | RP-cached PublicContext; re-loaded on restart |
